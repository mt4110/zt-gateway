# v0.1.0 Release Notes (Final Draft / 最終案)

## GitHub Release Title

`v0.1.0 - Zero-Trust Local Gateway 初回OSSリリース / Initial OSS Release`

## GitHub Release Body（日本語 / English MIX）

```md
## Overview / 概要

Zero-Trust local gateway monorepo の初回 OSS リリースです。

このプロジェクトは、まず「迷わないUX」を重視しています:

- Sender: `zt setup` -> `zt send`
- Receiver: `zt verify`

内部の `scan / sanitize / pack / verify` は `zt` がオーケストレーションします。  
そのため、untrusted network 上でも verification-first でファイル受け渡しできます。

## What’s Included / 同梱内容

- `zt` orchestration CLI (`setup`, `send`, `verify`, `doctor`)
- `secure-scan` integration (JSON mode + strict mode)
- `secure-rebuild` integration (initial rebuild/sanitize path)
- `secure-pack` integration (`.spkg.tgz` packet flow via `--client`)
- policy-driven extension enforcement (`DENY / SCAN_ONLY / SCAN_REBUILD`)
- local event spool + optional Control Plane sync
- Control Plane MVP (Go + Postgres optional dual-write)

## UX Highlights (v0.1.0) / 使い始めやすさ

- `zt setup` provides one-command onboarding checks（初期チェックを1コマンド化）
- `zt setup --json` supports support automation / diagnostics（`schema_version` 付き）
- `zt send --copy-command` copies the receiver `zt verify ...` command to clipboard（macOS優先）
- `zt send --share-format auto|ja|en` outputs receiver-facing share text in the preferred language
- `zt doctor` is a short alias for `zt config doctor`（短い運用導線）

## Security Notes / セキュリティ注意点

- Unknown extensions are denied by default (fail closed)
- `--strict` blocks degraded scan mode when scanners are unavailable
- Packet verification (`.spkg.tgz`) is stronger than the legacy `artifact.zp` transitional path
- Legacy `artifact.zp` verification remains transitional / PoC-level in parts

## Known Limitations / 既知の制約

- Scanner effectiveness depends on local availability of ClamAV / YARA / related tools
- `secure-rebuild` format coverage is still limited
- `zt send` without `--client` uses legacy PoC packer (transitional behavior)

## Quick Demo / 最短デモ

Sender:

```bash
zt setup
zt send --copy-command ./safe.txt
```

Receiver:

```bash
zt verify ./artifact.zp
```

## Upgrade / Migration Notes / 移行メモ

- Prefer `zt send --client <name>` for the newer packet (`.spkg.tgz`) flow
- `zt send --share-format auto` uses locale env (`LC_ALL` / `LC_MESSAGES` / `LANG`) to choose `ja` or `en`
- Use `zt --help-advanced` for full flags / operational modes
```
