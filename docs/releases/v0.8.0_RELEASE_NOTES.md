# v0.8.0 Release Notes

- Release date: 2026-02-26
- Scope: `zt send --share-json` の受信側レシート導線を additive 拡張

## Highlights

- `--share-json` payload に `receipt_hint` を追加
  - `receipt_hint.version` (`v1`)
  - `receipt_hint.path` (推奨レシート出力パス)
  - `receipt_hint.command` (`zt verify --receipt-out ...`)
- 既存の4フィールド（`kind`/`format`/`command`/`text`）は互換維持
- `send -> share-json -> verify -> receipt` の E2E 契約を強化
- v0.8.0 専用 CI ゲート `scripts/ci/check-v080-core-gate.sh` を追加

## Why this release

`zt verify --receipt-out` は既存実装だった一方、送信側 share payload から受信側へ「レシート保存つき verify 手順」を機械可読で渡せていませんでした。v0.8.0 はこのギャップを埋め、Slack/CI/ヘルプデスク導線での運用ミスを減らします。

## Compatibility

- Breaking change: なし
- 互換性ルール:
  - 既存4フィールドは不変
  - `receipt_hint` は additive field
  - 連携側は未知フィールド無視を継続可能

## Example (`--share-json`)

```json
{
  "kind": "receiver_verify_hint",
  "format": "en",
  "command": "zt verify -- './bundle_clientA_20260224T120000Z.spkg.tgz'",
  "text": "Please run the following command on the receiver side to verify the file.\nzt verify -- './bundle_clientA_20260224T120000Z.spkg.tgz'\n",
  "receipt_hint": {
    "version": "v1",
    "path": "./receipt_bundle_clientA_20260224T120000Z.json",
    "command": "zt verify --receipt-out './receipt_bundle_clientA_20260224T120000Z.json' -- './bundle_clientA_20260224T120000Z.spkg.tgz'"
  }
}
```
