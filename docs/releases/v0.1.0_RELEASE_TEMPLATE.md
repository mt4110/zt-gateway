# v0.1.0 Release Template

## Release Title (GitHub)

`v0.1.0 - Zero-Trust Gateway Monorepo (Initial OSS Release)`

## Release Notes Body (Template)

```md
## Overview

Initial OSS release of the Zero-Trust local gateway monorepo.

This release includes:

- `zt-gateway` orchestration CLI (`scan`, `send`, `verify`)
- `secure-scan` (TUI + JSON mode)
- `secure-pack` adapter integration for `.spkg.tgz` packet flow
- `secure-rebuild` (initial image rebuild path)
- policy-driven extension enforcement

## Highlights

### Added

- Monorepo structure for `zt-gateway / secure-pack / secure-scan / secure-rebuild`
- `zt` adapter layer for legacy/new tool coexistence
- `secure-scan check --json` and `--strict`
- `zt send --strict`
- extension policy file (`policy/extension_policy.toml`)
- packet smoke-test tooling (`scripts/dev/setup-secure-pack-localtest-gpg.sh`)

### Security Notes

- Default policy is fail-closed for unknown extensions
- `--strict` denies scan pass-through when no scanners are available
- Current verification for legacy `artifact.zp` remains PoC-level (packet verification is stronger)

## Known Limitations

- `secure-scan` effectiveness depends on local availability of ClamAV/YARA/ExifTool
- legacy `artifact.zp` verification path is transitional
- `secure-rebuild` format coverage is limited

## Upgrade / Migration Notes

- `zt send --client <name>` uses the new `secure-pack` packet path
- `zt send` without `--client` still uses legacy PoC packer (transitional)

## Verification

- Packet verification:
  - `zt verify <packet.spkg.tgz>`
- Legacy artifact verification:
  - `zt verify <artifact.zp>`
```

## Signed Tag Procedure (Checklist)

前提:

- GPG signing key configured for Git (`git config user.signingkey ...`)
- `gpg --list-secret-keys` で署名鍵が見える
- ワークツリーが整理済み

### 1. Final checks

```bash
go build ./gateway/zt
go build ./tools/secure-scan/cmd/secure-scan
go build ./tools/secure-pack/cmd/secure-pack
```

### 2. Update release docs

- `CHANGELOG.md` の `[Unreleased]` を整理
- `CHANGELOG.md` に `0.1.0` 日付を記載
- このテンプレートをベースに GitHub リリース本文を確定

### 3. Create signed tag

```bash
git tag -s v0.1.0 -m "v0.1.0"
git tag -v v0.1.0
```

### 4. Push commit and tag

```bash
git push origin <branch>
git push origin v0.1.0
```

### 5. Create GitHub release

- Tag: `v0.1.0`
- Title: 上記テンプレートのタイトル
