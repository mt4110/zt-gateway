# v0.9.0 Release Notes (Final Draft)

- Release date: 2026-02-26

## GitHub Release Title

- 日本語: `v0.9.0 - Relay Hook運用導線とShare JSONテンプレートを強化`
- English: `v0.9.0 - Strengthened Relay Hook Operations and Share JSON Templates`

## GitHub Release Body (日本語)

```md
## 概要

v0.9.0 では、Cloud-Agnostic Wrapper 連携と relay hook 実運用導線を強化しました。
送信側は Finder Quick Action から 1ファイル/複数ファイルをラップでき、受信側は `verify_command` / `receipt_command` に沿って検証を完了できます。

## 主な変更

- `zt send --share-json` に `channel_templates` を追加（additive）
  - `slack_text`, `email_subject`, `email_body`
  - `format=ja|en` でローカライズ
- `relay hook` 系コマンドを運用導線として強化
  - `relay hook wrap`
  - `relay hook serve` (`/v1/wrap`)
  - `relay hook finder-quick-action`
  - `relay hook install-finder` / `configure-finder`
- Finder Quick Action で `--force-public` を設定可能化（repo guard 方針に対応）
- Colima 前提の Ubuntu smoke 手順を明確化（Docker Desktop 非依存）

## 運用方針（repo guard / force-public）

- 現在の `mt4110/zt-gateway` は Public リポジトリのため、当面は Quick Action で `--force-public` を有効化
- 本番の推奨は private/internal リポジトリへ移行し、`--force-public` なしで fail-closed 運用

## 互換性

- Breaking change: なし
- 既存 `share-json` フィールド（`kind` / `format` / `command` / `text` / `receipt_hint`）は不変
- `channel_templates` は additive field のため、既存連携は未知フィールド無視で継続可能

## 検証

- `go test ./gateway/zt -count=1`
- `bash ./scripts/ci/check-v090-core-gate.sh`
- `bash ./scripts/ci/check-v080-core-gate.sh`
- `SKIP_ACTUAL_GATE=1 bash ./scripts/ci/check-pre-push-readiness.sh`
- Finder Quick Action 相当 runner で 1ファイル処理し、`verify_command` / `receipt_command` 実行を確認

## 参照

- `docs/architecture/V0.9.0_DESIGN.md`
- `docs/architecture/V0.9.0_IMPLEMENTATION_TICKETS.md`
- `docs/contracts/CLI_IO_DISPLAY_CONTRACT_v0.4.md`
- `docs/OPERATIONS.md`
```

## GitHub Release Body (English)

```md
## Overview

v0.9.0 strengthens Cloud-Agnostic Wrapper integration and relay hook operational paths.
Senders can wrap files from Finder Quick Action (single or multiple files), and receivers can complete validation using `verify_command` / `receipt_command`.

## Highlights

- Added additive `channel_templates` to `zt send --share-json`
  - `slack_text`, `email_subject`, `email_body`
  - localized by `format=ja|en`
- Strengthened `relay hook` operational flow
  - `relay hook wrap`
  - `relay hook serve` (`/v1/wrap`)
  - `relay hook finder-quick-action`
  - `relay hook install-finder` / `configure-finder`
- Enabled `--force-public` in Finder Quick Action flow for repo-guard policy alignment
- Clarified Colima-first Ubuntu smoke procedure (without Docker Desktop dependency)

## Operational Policy (repo guard / force-public)

- Since `mt4110/zt-gateway` is currently public, keep `--force-public` enabled for this repository
- Production recommendation: move to a private/internal repository and run fail-closed without `--force-public`

## Compatibility

- Breaking change: none
- Existing `share-json` fields (`kind` / `format` / `command` / `text` / `receipt_hint`) remain unchanged
- `channel_templates` is additive, so existing integrations can continue by ignoring unknown fields
```
