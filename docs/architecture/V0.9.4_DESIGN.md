## 2. 残機能の全量棚卸し

## P0: True Zero Trust Core（v0.9.4）

### G-9401 Rebuild 真実性ギャップ（最優先）

- `secure-rebuild` は画像以外で identity copy を実行してしまう。
- `SCAN_REBUILD` 指定でも実質 sanitize されない経路がある。
- Evidence:
  - `tools/secure-rebuild/main.go` (`No specific sanitizer found. Performing identity copy.`)
  - `gateway/zt/commands_flow.go`（`SCAN_REBUILD` 時に `secure-rebuild` 実行）

### G-9402 SCAN_REBUILD対象と実装対応の不整合

- confidential/regulated の policy は `.pdf/.docx/.xlsx/.pptx` を `scan_rebuild_extensions` に含むが、rebuild 実装が追随していない。
- Evidence:
  - `policy/profiles/confidential/extension_policy.toml`
  - `policy/profiles/regulated/extension_policy.toml`
  - `tools/secure-rebuild/main.go`

### G-9403 setup 実結果と CI gate 判定の乖離

- actual gate は `gate_ok` を pass/fail 基準にし、`setup_ok=false` は informational 扱い。
- true-zt では trust-critical fail を release gate で止める必要がある。
- Evidence:
  - `scripts/ci/check-zt-setup-json-actual-gate.sh`
  - `scripts/ci/check-pre-push-readiness.sh`

### G-9404 signer pin bootstrap の運用標準化不足

- root pin は bootstrap 導線がある一方、verify signer pin は初期未設定で fail になりやすい。
- Evidence:
  - `go run ./gateway/zt setup --json`（`team_boundary_signer_pin_consistency=fail`）
  - `gateway/zt/commands_verify.go`（signer pin missing/mismatch fail-closed）

### G-9405 Team Boundary の production default 不在

- 既定 policy が `enabled=false` のため、境界強制が無効でも運用開始できる。
- true-zt 運用向けには「有効化テンプレ + 強制モード」が必要。
- Evidence:
  - `policy/team_boundary.toml`

### G-9406 Control Plane ingest の envelope 必須化未固定

- 検証鍵レジストリ未有効時は raw JSON ingest を許容。
- true-zt 運用では signed envelope を既定必須に寄せる必要がある。
- Evidence:
  - `control-plane/api/cmd/zt-control-plane/ingest_envelope.go`

### G-9407 Control Plane 認証の fail-open 設定余地

- `ZT_CP_API_KEY` 未設定時は API key チェックが無効化される。
- true-zt 運用では auth 未設定起動を禁止すべき。
- Evidence:
  - `control-plane/api/cmd/zt-control-plane/main.go`
  - `control-plane/api/cmd/zt-control-plane/ingest_envelope.go`

### G-9408 Rebuild 証跡の不足（監査再現性）

- 現状の artifact event は packet hash中心で、rebuild前後差分や sanitizer情報が不足。
- 監査時に「何で sanitize したか」の機械再現性が弱い。
- Evidence:
  - `gateway/zt/events_emit.go`
  - `gateway/zt/commands_flow.go`

### G-9409 仕様文書の現在地乖離

- README/THREAT_MODEL に旧ギャップ文言（統合途中・verify再設計中）が残り、実装状態とズレる。
- true-zt 判定基準を文書上で一意化する必要がある。
- Evidence:
  - `README.md`（現在の穴・未完了の点）
  - `THREAT_MODEL.md`（draft/pre-v0.1.0 の未緩和記述）

## 3. v0.9.4 Scope（確定提案）

v0.9.4 は dashboard を入れず、P0 を閉じる release に固定する。

### C-9401 Rebuild Truthfulness Contract

- `SCAN_REBUILD` 対象で unsupported sanitizer は identity copy を禁止し fail-closed。
- 実 sanitize 成功時のみ pack 継続。

### C-9402 Profile-Rebuild Consistency Contract

- `scan_rebuild_extensions` と sanitizer 対応表を契約化し、齟齬を setup/doctor で fail。

### C-9403 Strict Actual Gate Contract

- `check-zt-setup-json-actual-gate.sh` に strict mode を追加。
- strict mode では trust-critical fail（root pin / signer pin / boundary必須時設定欠落）を gate fail。

### C-9404 Signer Pin Bootstrap Contract

- signer pin の配布/検証 bootstrap script を追加し、root pin と同等に標準化。

### C-9405 Team Boundary Production Mode Contract

- `ZT_REQUIRE_TEAM_BOUNDARY=1` で `enabled=false` を fail-closed。
- production テンプレ policy を別ファイルで提供。

### C-9406 Signed Envelope Required-by-Default Contract

- CP ingest は既定で envelope required。
- raw JSON許容は明示的な開発モードのみ。

### C-9407 CP Auth Mandatory Contract

- production mode 起動時に API key/mTLS のどちらか必須。

### C-9408 Rebuild Provenance Contract

- artifact/audit に rebuild method, input/output hash, sanitizer version を追加。

### C-9409 Documentation Truth Contract

- README/THREAT_MODEL/OPERATIONS の保証境界を 2026-02-27 実装状態へ更新。

## 6. Exit Criteria（v0.9.4）

- `SCAN_REBUILD` 対象で sanitize 未対応形式のすり抜けが 0 件（fail-closed）
- actual gate strict mode で trust-critical 欠落が必ず fail
- signer pin / root pin の bootstrap が CI/ローカル両方で一発収束
- CP ingest が production mode で signed envelope + auth を必須化
- rebuild provenance が audit 再現可能な粒度で残る
- v0.9.4 専用ゲートが pre-push に統合される
