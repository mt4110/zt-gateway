# v0.9.7 Design: Dashboard Alert Dispatch Safety Hardening

## 0. Baseline

- 確認日: 2026-02-27
- ブランチ: `feat/v0.9.7`
- 現状:
  - dashboard は `control_plane` / `kpi` / `incidents` / `alerts` を同一JSONで返す
  - 外部通知実行 (`/api/alerts/dispatch`) は fail-closed（HTTPS + allowlist + opt-in）で拒否できる
- 残ギャップ:
  - `ZT_DASHBOARD_ALERT_DISPATCH_ENABLED=1` かつ `ZT_DASHBOARD_ALERT_WEBHOOK_ALLOW_HOSTS` 未設定の「運用上の危険状態」が `danger` に即時可視化されない
  - 当該ギャップを固定する専用ゲートがない

## 1. Goal / Non-Goal

### Goal

- 危険な通知設定を dashboard snapshot の `danger` / `alerts` に deterministic に載せる
- 契約テストと専用CI gateで回帰を独立検知する

### Non-Goal

- 通知チャネル追加（Slack/Discord/LINE 以外）
- webhook 送信処理本体のプロトコル変更
- Control Plane dashboard API 仕様変更

## 2. Contract Scope (C-9701..C-9703)

### C-9701 Unsafe Dispatch Config Visibility Contract

- 条件:
  - `ZT_DASHBOARD_ALERT_DISPATCH_ENABLED=1`
  - `ZT_DASHBOARD_ALERT_WEBHOOK_ALLOW_HOSTS` が空
- 結果:
  - `danger.level=high`
  - `danger.signals[]` に `dashboard_alert_dispatch_unsafe_config` が必ず含まれる

### C-9702 Safe Config Non-Regression Contract

- `ZT_DASHBOARD_ALERT_WEBHOOK_ALLOW_HOSTS` が設定済みの場合、`dashboard_alert_dispatch_unsafe_config` を誤検知しない

### C-9703 v0.9.7 Dedicated Gate Contract

- `scripts/ci/check-v097-dashboard-safety-gate.sh` を追加し、v0.9.7契約だけを単体実行できる
- `check-pre-push-readiness.sh` へ組み込み、回帰を独立検知する

## 3. Implementation Tickets

1. TKT-9701 `collectDashboardSnapshot` に dispatch guardrail signal 追加

## 4. Exit Criteria

- unsafe 設定時に `dashboard_alert_dispatch_unsafe_config` が必ず出る
- allowlist 設定時に誤検知しない
- v0.9.7 gate が pre-push へ統合される
