# v0.9.3 Design (Draft): Remaining Gap Closure

## 0. Baseline

- 確認日: 2026-02-27
- `feat/v0.9.3` 開始時点で `check-v092-boundary-gate.sh` は green
- v0.9.2 までで team boundary 契約は成立したが、以下の残タスクが運用上の抜け道として残る
  - `secure-pack receive` が signer allowlist 強制なしで復号・展開できる経路を持つ
  - `--allow-degraded-scan` により no-scanner-available を許容できる経路が残る
  - `zt setup --json` / `zt config doctor --json` で scan posture の弱化（運用設定劣化）を十分に fail-closed 検知できない
  - v0.9.3 専用の独立ゲートが未整備

## 1. Goal / Non-Goal

### Goal

- 受信経路（verify / receive）で signer trust anchor 強制を完全に一致させる
- 境界運用時の degraded scan を break-glass 以外で通らないよう固定する
- setup/doctor で scan posture 劣化を deterministic に検知できるようにする
- v0.9.3 契約のみを独立実行できる CI gate を追加する

### Non-Goal

- `secure-scan` エンジン自体の検知ロジック刷新
- 新しい外部SaaS連携や管理UI追加
- CDR対応形式の全面拡大（v1.x テーマ）

## 2. 残タスク棚卸し（Evidence付き）

### G-9301: receive 経路の signer allowlist 未強制

- `tools/secure-pack/internal/workflows/workflows.go` の `ReceiverWorkflow` は `pack.UnpackPacket` を直接呼ぶ
- `pack.UnpackPacket` 側には signer allowlist 未実装 TODO が残る
- 結果として `secure-pack verify` と `secure-pack receive` で trust anchor が不一致

### G-9302: degraded scan override の境界運用ガード不足

- `zt send --allow-degraded-scan` は strict を無効化できる
- team boundary active 時でも degraded override の禁止/制限が契約化されていない

### G-9303: scan posture 劣化の doctor 検知不足

- `required_scanners` 空や `require_clamav_db=false` を profile/boundary 文脈で十分にブロックできない
- 劣化設定が残ったまま運用に入ると、no-scanner-available を許容しやすい

### G-9304: v0.9.3 回帰を独立検知する gate 不在

- 既存は v0.9.2 までの boundary gate で、v0.9.3 固有契約の検知粒度がない

## 3. Contract Scope (C-9301..C-9305)

### C-9301 Verify/Receive Trust-Anchor Parity Contract

- `secure-pack receive` は `secure-pack verify` と同一の signer pin 解決・照合を必須化する
- signer pin 未設定/不一致/形式不正時は extraction 前に fail-closed
- 受信成功時に `SIGNER_FINGERPRINT` を証跡として出力可能にする

### C-9302 Unpack Path Guard Contract

- `pack.UnpackPacket` の「署名検証のみ」依存経路を廃止または internal 限定化する
- 受信ワークフローは「verify(with signer pin) 成功後のみ decrypt/extract」を契約化する
- 未検証 extraction を許さない

### C-9303 Boundary Degraded-Scan Guardrail Contract

- team boundary active 時は `--allow-degraded-scan` を fail-closed（既定）
- 例外は break-glass token あり + policy guardrail適合時のみ許可
- 理由不足/期限切れ/承認者不足は既存 break-glass エラー体系で即時失敗

### C-9304 Scan Posture Doctor Contract

- `zt setup --json` / `zt config doctor --json` に scan posture 専用チェックを追加
- 追加チェック例:
  - `scan_posture_required_scanners_non_empty`
  - `scan_posture_clamav_db_required_for_strict_profiles`
  - `scan_posture_boundary_degraded_override_blocked`
- fail 時は remediation を返し、overall を fail-closed 側に寄せる

### C-9305 v0.9.3 CI Gate Contract

- `scripts/ci/check-v093-remaining-gap-gate.sh` を追加
- `check-pre-push-readiness.sh` に組み込み、v0.9.3 契約を独立回帰検知する

## 5. Recommended Order

1. TKT-9301 Receive parity
2. TKT-9302 Unpack hardening
3. TKT-9303 Boundary degraded guardrail
4. TKT-9304 Setup/doctor posture checks
5. TKT-9305 CI gate

## 6. Exit Criteria

- `secure-pack verify` と `secure-pack receive` の signer trust anchor 判定が一致
- allowlist 未設定/不一致 packet は extract されない
- team boundary active で degraded scan override が通常経路で通らない
- setup/doctor で scan posture 劣化が fail-closed で検知される
- v0.9.3 gate が pre-push に統合され、回帰が独立検知される
