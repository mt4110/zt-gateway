# v0.7.0 Implementation Tickets (C-7001..C-7005)

本ファイルは `docs/architecture/V0.7.0_DESIGN.md` の Contract Matrix を、実装着手可能な粒度へ分割したチケット一覧です。

## TKT-7001: File Type Guard Explainability (`C-7001`)

- Contract:
  - `policy_magic_mismatch` 時に `file_type_guard.reason_code` を JSON 出力へ固定
- 変更対象:
  - `gateway/zt/file_type_guard.go`
  - `gateway/zt/commands_flow.go`
  - `gateway/zt/events_emit.go`（または policy decision/event 生成箇所）
- 実装タスク:
  1. `enforceFileTypeConsistency` のエラーを構造化情報へ変換できるようにする
  2. `zt send` block 時の policy decision JSON に `file_type_guard` を追加
  3. event payload へ同一構造を載せる
- テスト:
  - `TestFileTypeGuard_JSONExplainabilityContract`
  - 既存 `file_type_guard_test.go` で reason_code アサーション追加
- Definition of Done:
  - mismatch 系ケースで `reason_code` 欠落ゼロ
  - 既存契約テスト回帰なし

## TKT-7002: Scan Posture Contract (`C-7002`)

- Contract:
  - `strict_effective` / `allow_degraded_scan` / scanner requirement を JSON で可視化
  - strict profile で degraded 混入時は `policy_scan_posture_violation`
- 変更対象:
  - `gateway/zt/commands_flow.go`
  - `gateway/zt/scan_policy.go`
  - `gateway/zt/policy_decision.go`
  - `gateway/zt/error_codes.go`（必要なら追加）
- 実装タスク:
  1. 実効 scan posture を算出する構造体を追加
  2. send/scan JSON 出力に posture を追加
  3. strict profile 逸脱時の fail-closed エラーコードを固定
- テスト:
  - `TestScanPosture_ProfileContract`
  - profile ごとの strict/degraded 許容境界テスト
- Definition of Done:
  - `public` 以外で degraded 許容が通らない
  - posture フィールドが JSON で常時取得可能
