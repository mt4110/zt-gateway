# Enterprise Usage Guide: Secure-Scan

[English Version](./en/ENTERPRISE_GUIDE.md)

**Target Audience**: Security Operations, Field Engineering Teams, Compliance Officers.
**Version**: 1.0.0
**Confidentiality**: Internal Use Only (Unless sanitized).

---

## 1. Executive Summary

`secure-scan` は、組織内の機密データおよび成果物を、外部への**データ流出リスクをゼロに保ちながら**検査するために設計された、ローカル完結型のセキュリティツールです。
クラウドベースのウイルス対策ソフトとは異なり、ファイルをインターネット上のサーバーにアップロードすることなく、ローカル環境（PC上）でウイルス検知、機密情報の発見、およびメタデータの除去を行います。

## 2. Security Architecture & Trust

### 2.1 ローカル完結の原則 (Local Execution)
本ツールの最大の特徴は、**「すべての処理がローカルメモリ上で完結する」**点です。
*   **No Egress**: スキャン対象のファイルデータが外部ネットワーク（SaaS等のAPI）に送信されることは一切ありません。
*   **Memory Safety**: Go言語で実装されており、バッファオーバーフロー等のメモリ脆弱性リスクが最小化されています。
*   **Isolated Environment**: Nix技術により、実行に必要なライブラリ（ClamAV等）はシステムから隔離されたサンドボックス環境で動作し、ホストOSを汚染しません。

### 2.2 採用モジュールの信頼性 (Module Provenance)
本ツールは、世界的なデファクトスタンダードとして長年の実績を持つエンジンを採用しています。

## 3. Scope & Limitations (対策範囲と限界)

現場での利用において、「何ができて、何ができないか」を理解することは信頼の鍵となります。

### ✅ できること (Covered)
*   **既知のマルウェア検知**: 数百万のシグネチャデータベースに基づくウイルス検知。
*   **静的解析**: ファイルを実行せずに中身をスキャンするため、スキャン自体による感染リスクがありません。
*   **深層スキャン**: 多重に圧縮されたアーカイブ（Zip in Tar in Zipなど）の展開スキャン。
*   **メタデータ除去**: 撮影地(GPS)や作成者名など、ファイルそのものの価値を損なわずにプライバシー情報のみを削除。

### ⚠️ できないこと (Limitations)
*   **ゼロデイ攻撃の即時遮断**: 未知のウイルス（シグネチャ未登録）は検知できません。振る舞い検知（EDR）ではないためです。
*   **動的解析 (Sandbox)**: 「実行したとき何が起きるか」の検証は行いません。
*   **クラウド並みの更新速度**: 定義ファイルはダウンロード時点のものを使用するため、クラウドリアルタイム検知に比べ、数十分〜数時間のラグが発生する可能性があります。

## 4. Operation Guidelines (運用ガイドライン)

### 4.1 前提条件: Private Repository Only
本ツールは、**組織管理下のプライベートリポジトリまたはローカルディレクトリ**での利用を前提としています。
誤って公開された環境（パブリックリポジトリ等）で実行し、意図せず内部情報を扱うリスクを防ぐため、**「Private Repo Guard」**機能が実装されています。

*   **動作**: 実行ディレクトリの `git remote` を確認し、パブリックなホストや組織外のリポジトリであると判定された場合、起動をブロックします。

### 4.2 更新運用 (Update Policy)
脅威情報は日々変化します。利用前には必ず以下の手順で「鮮度」を保ってください。

```bash
# 1. ツール定義の更新 (毎日推奨)
secure-scan update

# 2. スキャンの実行
secure-scan check ./project-dir
```

### 4.3 脆弱性・インシデント対応
万が一、本ツールの依存ライブラリ（ClamAV等）に脆弱性が見つかった場合：
1.  **Nix Flakeの更新**: 管理チームが直ちに `flake.lock` を更新し、PRを発行します。
2.  **即時適用**: ユーザーは `nix develop` コマンドを打つだけで、パッチ適用済みの安全なバイナリに切り替わります。

---
### 4.4 High Availability & Freshness (98% Assurance)
本ツールは、バックグラウンドでの**「10分間隔の自動定義更新」**を運用標準とすることで、ローカルスキャンでありながらクラウド検知に近い鮮度（理論値 98%以上のカバー率）を維持します。
ただし、定義更新のタイムラグが数分発生しうる性質上、ゼロデイ攻撃への完全な即応性（残り2%）はEDR等の振る舞い検知で補完することを推奨します。
